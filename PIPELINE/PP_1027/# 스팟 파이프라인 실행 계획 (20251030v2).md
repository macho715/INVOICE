## 실행 계획

### 1단계: 파이프라인에 새 파일 지정

**작업**: pipeline_config.yaml을 임시로 수정하거나 파이프라인 실행 시 직접 파일 경로 지정

**방법 1**: 파이프라인 실행 시 파일 경로 직접 지정 (권장)

```bash
python run\run_pipeline.py --stage 1 --master-file "data/raw/HITACHI/Case List(20251030)v2.xlsx" --warehouse-file "data/raw/HITACHI/HVDC WAREHOUSE_HITACHI(HE)20251030v2.xlsx"
```

**방법 2**: config 파일 임시 수정

- `config/pipeline_config.yaml`의 stage1.io 섹션 수정
  - master_file: `data/raw/HITACHI/Case List(20251030)v2.xlsx`
  - warehouse_file: `data/raw/HITACHI/HVDC WAREHOUSE_HITACHI(HE)20251030v2.xlsx`

### 2단계: 파이프라인 실행

**작업**: Stage 1 실행 (다중 시트 동기화)

**명령어**:

```bash
python run\run_pipeline.py --all
```

또는 Stage 1만:

```bash
python run\run_pipeline.py --stage 1
```

**예상 동작**:

- Case List(20251030)v2.xlsx의 10개 시트 모두 처리
- 각 시트를 Warehouse의 "Case List, RIL" 시트와 매칭하여 동기화
- 매칭되지 않는 시트는 master-only로 처리

### 3단계: 결과 확인

**확인 사항**:

- Stage 1 출력 파일의 시트 수 (10개 시트 모두 반영 여부)
- 각 시트별 동기화 통계
- 최종 merged 파일에 모든 시트 데이터 포함 여부

**출력 파일**:

- `data/processed/synced/HVDC WAREHOUSE_HITACHI(HE).synced_v3.4.xlsx` (multi-sheet)
- `data/processed/synced/HVDC WAREHOUSE_HITACHI(HE).synced_v3.4_merged.xlsx` (merged)

## 주의사항

1. **pipeline config 무시**: 실행 시 직접 파일 경로 지정
2. **다중 시트 처리**: Stage 1은 multi-sheet를 지원하므로 10개 시트 모두 처리 가능
3. **시트 매칭**: 파이프라인은 semantic matching으로 시트를 매칭함

   - "Case List, RIL" ↔ "Case List, RIL" (정확 매칭)
   - 나머지 시트들은 warehouse에 없으므로 master-only로 처리될 수 있음

## 예상 결과

- Case List, RIL: Warehouse의 기존 "Case List, RIL"과 동기화
- 나머지 9개 시트: Master-only로 처리되어 merged 파일에 포함
